Why use a token?

The Storjcoin X (or SJCX for short) was a concept that we introduced in 2014: a digital token that can be used within the Storj ecosystem to purchase additional cloud-based storage space. Users with extra hard drive space can sell it to other users in exchange for SJCX. If a user needs to buy cloud storage space, they can do so with SJCX too. There are many reasons why we chose to build our own token, instead of using a more “traditional” cryptocurrency:

Micropayments

The Storj ecosystem requires micropayments to stay healthy. Unfortunately, established cryptocurrencies such as Bitcoin and Ether are not divisible enough to allow such payments. Additionally, we would need to lock up funds in channels to conduct micropayments, and operators usually charge for this service. The SJCX allows us to deal with both of these issues and set up a payment system that suits the Storj ecosystem perfectly.

Bootstrapping

When we first started the network, Storj was an empty market. No farmers, no renters, and nothing to buy or sell. Fortunately (now) and unfortunately (then), Storj only works at scale. Using SJCX to bootstrap the Storj network let us kickstart the system. It helped us identify and fix problems at scale, and paved the way for a larger, more stable network. The new token (under a new ticker: STORJ) will further help us to quickly develop the Storj ecosystem in a community oriented fashion.

Flexibility

Technology develops and changes over time. Using our own token allows us to organically change with it, without being locked into a particular technology. For example, we are now migrating SJCX to the Ethereum platform, in part to fully harness the technology’s Smart Contract capabilities, which were previously unavailable to us.

Stability

Cryptocurrency markets are volatile. A separate token provides a layer of protection from the volatility of larger markets. Volatility is extremely dangerous for long-term storage contracts. Although no token can be completely safe from the ebbs and flows of the largest markets, they can be more stable in comparison.

Growth

Storj is here for the long haul. Using our own technology that we can expand and adapt as needed makes us better prepared for the future. We can use SJCX for projects that we could not pursue using just Bitcoin or Ether as the backbone.

Ecosystem

Using STORJ allows us to migrate to the accepted standard, without being locked into a particular technology.

How will Storj change after the token sale?

For users

Improved User Experienced

The cloud storage of the future needs to be accessible to everyone. That’s why we will be investing a lot of time and resources to make Storj user-friendly. We will be announcing a partnership for this project shortly, so stay tuned!

Increased functionality and more features

The basic functionality of cloud storage (storing and retrieving files) is already available to our users. We have many more use cases in the roadmap that are currently under development.

Smoother, simpler payments

As the number of users buying and selling storage space grows, we will need to implement real-time payments with micropayment channels. We’re working with industry leaders to make this happen.

For the Storj Community

More developer tools

We have already released a functioning API, CLI and GUI for developers to use to push, pull and store data, but in the future we intend to integrate our data layer tool with other developer ecosystem tool, similar to how we integrated with Heroku for a one-click implementation for developers.

Improved token supply balance

As part of the upcoming migration plan, we want to create a better balance between the Storj Community and Storj Labs. We will shortly be releasing a solution to resolve the token supply imbalance and to create a shared foundation for the long term success of the ecosystem.

Community self-organization

Over the next few months we will be working with community leaders to build processes and tools that will help the community self-organize. We will give the community access to things like documentation and apps, to enable everyone to build more awesome things with us. We also plan to provide the community with a voice on how company resources are allocated.

For Storj Labs

Smart Contact Integration

Blockchain-based smart contracts cannot manage distributed storage at scale by themselves, but used together with the Bridge model they can be very useful. We’re also exploring on-chain structures that will allow the next generation of applications on Ethereum to use Storj as a native data layer, as well as smart-contract enabled economic reputation and bonding systems.

Federated Bridges

Storj Bridges provide a set of decentralized access points to the network. Each Bridge helps manage the state of files that are uploaded. This binds clients to specific Bridges. To alleviate this drawback, in the future Storj will take advantage of audit-verification smart contracts to create a federated file management system distributed across a group of Bridges.
